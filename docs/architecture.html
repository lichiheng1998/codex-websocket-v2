codex-websocket-v2 Architecture

Hermes Agent plugin — bridge sessions to OpenAI Codex app-server via WebSocket JSON-RPC

Per-Session Isolation

  • • Each chat gets its own CodexSession
  • • Independent WebSocket + event loop + thread
  • • Own task map and config (model, mode)
  • • Session key: platform:chat_id

Shared app-server

  • • Single codex app-server subprocess
  • • Ref-counted via ServerManager singleton
  • • First session spawns, last exit terminates
  • • Port auto-selected on startup

Cross-Loop Notification

  • • Codex runs on per-session asyncio loop
  • • Platform adapters (WeChat) on main loop
  • • notify.py bridges via run_coroutine_threadsafe
  • • set_main_loop() at plugin register time
